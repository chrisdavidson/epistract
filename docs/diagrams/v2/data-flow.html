Epistract v2.0 — Pipeline Data Flow

Document corpus → per-document parallel extraction → sift-kg graph build → domain epistemic overlay → artifacts

Per-Document Parallelism

  • • 1 epistract:extractor subagent per document
  • • Each reads SKILL.md + applies domain prompts independently
  • • Scales to 30+ concurrent Claude subagents in practice
  • • Isolated scope — no cross-document context leakage
  • • Emits Pydantic-shaped DocumentExtraction JSON

sift-kg Graph Construction

  • • Canonical-name dedup (capital-C → CamelCase)
  • • SemHash fuzzy matching for variants
  • • Louvain community detection + LLM narration
  • • Provenance preserved per edge (source doc + evidence)
  • • MultiDiGraph — supports multiple edges between nodes

Two-Layer Output

  • • Layer 1 — nodes + edges extracted from documents
  • • Layer 2 — claims.json with domain-specific status/severity
  • • Workbench renders both simultaneously
  • • Artifacts are the contract between pipeline and consumers
  • • /epistract:query + /epistract:ask read the same JSON
